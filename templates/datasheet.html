Purpose & Scope

The DNV–Bastion Coordinator is an internal web-based compliance management system purpose-built to track Bastion Technologies' document submission workflow against DNV GL qualification requirements for Sentinel 001 (Vanguard 02) under DNV-RU-UWT Pt.5 Ch.10 Sec.1. It manages the full lifecycle from DNV document requirement definition through Bastion document submission, multi-round DNV comment exchange, SLA tracking, and final qualification acceptance.

Application Modules
Dashboard
/    /exec
Live KPI summary — submissions, TQ log, action tracker, audit readiness, DocReq coverage. Executive management view auto-refreshes every 2 min.
Doc Requirements (DocReq)
/docreq
Master-detail compliance tracker. 139 DNV requirements seeded from DNV-RU-UWT. Links Bastion documents, tracks multi-round comment threads with 10bd SLA enforcement.
Submissions Coordinator
/coordinator
Full submission register — document status, DNV workflow state, revision tracking, file attachments, PDM import via SolidWorks CSV export.
TQ Log
/coordinator → TQ tab
Technical Query register with SLA forecast. Tracks open / closed TQs, response age, breach risk. PDF export with forecast chart.
Action Tracker
/coordinator → Actions tab
Open action items with owner, due date, priority, and status. Amber/red ageing indicators. Filterable by owner and status.
Reports
/reports
Four PDF-exportable report tabs: SLA Breach Forecast, Interface Health Scorecard, Audit Readiness, DNV Doc Requirements Coverage.
Kanban Board
/kanban
Visual workflow board for document states. Drag-and-drop card management. Filtered by document type and owner.
SLA Digest
sla_digest.py · /api/admin/sla-digest
Daily Teams webhook notification. Posts breach (10+ bd) and warning (8+ bd) open threads to configured MS Teams channel. Schedulable via Windows Task Scheduler.
